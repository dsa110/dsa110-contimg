    Temporal Flagging Tracking SystemÂ¶

    Date: 2025-11-19
    Author: AI Agent + Dana
    Type: Implementation Guide
    Status: âœ… Complete
    Related: SPW Flagging Process

    Executive SummaryÂ¶

    The DSA-110 pipeline now includes temporal flagging tracking that captures flag states at each critical phase of calibration. This solves a critical diagnostic problem: determining with certainty (not inference) why specific SPWs fail calibration.

    Before: "SPW 9 is 100% flagged, likely due to RFI" (guessing)
    After: "SPW 9: Pre-cal 85.2% flagged, refant 100% flagged â†’ calibration solve failed â†’ applycal flagged remaining 14.8%" (certain)

    The Problem This SolvesÂ¶

    Original IssueÂ¶

    When examining a calibrated MS, we observed:

    The following MS spws have no corresponding cal spws in calibration tables: 9 14 15

    Checking the final MS state showed SPWs 9, 14, 15 were 100% flagged. But we couldn't determine when this happened:

    • Were they 100% flagged BEFORE calibration (Phase 1 RFI flagging)?
    • Did they have partial flagging, causing calibration to fail, then applycal flagged the rest?
    • Was it reference antenna issues, or overall data quality?

    Root Cause of UncertaintyÂ¶

    The diagnostic script initially examined the final MS state (post-applycal) and concluded SPWs were "100% flagged pre-calibration." This was circular logic:

    1. Observed: SPWs are 100% flagged (in final state)
    2. Concluded: They were 100% flagged pre-calibration
    3. Explained: That's why calibration failed
    4. Result: applycal flagged them 100%

    But actually, the SPWs may have been partially flagged (60-95%) pre-calibration, causing calibration to fail, then applycal flagged the remaining data.

    Key InsightÂ¶

    Temporal ordering matters. You cannot infer pre-calibration state from post-applycal state because applycal modifies the flag column for SPWs without calibration solutions.

    Findings & Lessons LearnedÂ¶

    Key FindingsÂ¶

    1. SPWs 9, 14, 15 Failure Cause (Definitive):
    2. Pre-calibration flagging: 100% (all data flagged before calibration)
    3. Reference antenna 103: 100% flagged in these SPWs
    4. Calibration tables: NO SOLUTIONS confirmed

    5. Root cause: Refant 103 fully flagged during RFI flagging (Phase 1)

    6. Other SPWs (0-8, 10-13) Behavior:

    7. Pre-calibration flagging: 5-93% (per-channel, partial)
    8. Calibration succeeded for these SPWs
    9. Post-applycal flagging: Similar to pre-calibration (no significant change)

    10. Result: Calibration applied successfully, existing flags preserved

    11. CASA applycal Behavior:

    12. Confirmed: applycal DOES NOT flag data when calibration exists
    13. Confirmed: applycal DOES flag 100% when NO calibration solution exists
    14. This is CASA's documented behavior: "flag data for which there is no calibration available"

    Lessons LearnedÂ¶

    1. Temporal Ordering is Critical
    2. Cannot infer pre-state from post-state when intermediate steps modify data
    3. Must capture snapshots at each phase for definitive analysis

    4. Retrospective analysis has fundamental limitations

    5. The "Likely" Problem

    6. Hedging language ("likely", "probably", "appears to be") indicates inference, not certainty
    7. Inference from indirect evidence can be wrong

    8. Always prefer direct examination of data at appropriate time points

    9. Circular Logic Trap

    10. Observing: "SPWs are 100% flagged"
    11. Concluding: "They were 100% flagged pre-calibration"
    12. Actually: They may have been partially flagged, then applycal flagged the rest

    13. Solution: Capture pre-calibration state independently

    14. Reference Antenna Critical

    15. Calibration cannot proceed if refant has no valid data in an SPW
    16. Even partial SPW flagging is acceptable if refant has valid data

    17. Refant selection strategy should consider per-SPW flagging

    18. Per-Channel vs Per-SPW Flagging

    19. Pre-calibration: Per-channel flagging (preferred, preserves good data)
    20. Post-calibration-failure: Full SPW flagging (automatic CASA behavior)
    21. This is by design and appropriate

    Documentation QualityÂ¶

    The original diagnostic script was logically sound but had a fundamental assumption error:

    • Assumed: MS flag state represents pre-calibration state
    • Actually: MS flag state represents post-applycal state
    • Impact: "Definitive" conclusions were based on wrong data

    Fix: Capture snapshots at each phase during pipeline execution, not retrospective examination.
