    Transit Search Process - Step-by-Step Explanation¶

    Overview¶

    The transit search process finds when a calibrator source (e.g., "0834+555") transits the meridian and matches it to available observation data. This is used to create mosaics centered on calibrator transits.

    High-Level Flow¶

    1. User requests mosaic for calibrator (e.g., "0834+555")
2. Calculate theoretical transit times
3. Search for observation groups near transit times
4. Validate transit matches (time + declination + primary beam)
5. Select best matching group
6. Create time window centered on transit
7. Find MS files in that window

    Step-by-Step Details¶

    Step 1: Calculate Theoretical Transit Times¶

    Location: dsa110_contimg/calibration/schedule.py

    Function: previous_transits(ra_deg, start_time, n)

    Process:

    1. Input: Calibrator RA (right ascension) in degrees
    2. Calculate next transit: Uses next_transit_time() which:
    3. Converts RA to hours
    4. Gets Local Sidereal Time (LST) at start time
    5. Calculates when LST = RA (meridian transit)
    6. Iteratively refines to find exact transit time
    7. Step backward: Steps back in 1 sidereal-day increments to get previous transits
    8. Output: List of Time objects representing transit times

    Key Formula:

    Transit occurs when: LST = RA (modulo 24 hours)
Sidereal day = 1.0 / SIDEREAL_RATE ≈ 0.99727 solar days

    Example:

    • Calibrator "0834+555" has RA ≈ 8.58 hours
    • Transit occurs when LST = 8.58 hours
    • Previous transits are 1 sidereal day apart

    Step 2: List Available Transits with Data¶

    Location: dsa110_contimg/conversion/calibrator_ms_service.py

    Function: list_available_transits(calibrator_name, max_days_back)

    Process:

    1. Load calibrator coordinates:
    2. RA and Dec from catalog (e.g., VLA catalog)

    3. Example: "0834+555" → RA=8.58°, Dec=55.57°

    4. Calculate search window:

    5. Start: Time.now() - max_days_back days
    6. End: Time.now()

    7. Cutoff: Filter out groups older than cutoff

    8. Query database for complete groups:

    9. Query products_db for 16-subband groups in time range
    10. Groups must be complete (all 16 subbands present)

    11. Uses query_subband_groups() function

    12. Get theoretical transits:

    13. Calls previous_transits(ra_deg, start_time=Time.now(), n=max_days_back)

    14. Gets list of transit times going back max_days_back days

    15. Match transits to groups:

    16. For each transit, find groups whose observation window contains the transit
    17. Calls _process_transits_for_available_data()

    Output: List of transit info dicts, sorted by most recent first:

    {
    'transit_iso': '2025-11-07T13:19:23.000',
    'transit_mjd': 60345.555,
    'group_id': '2025-11-07T13:19:23',
    'group_mid_iso': '2025-11-07T13:19:25.000',
    'delta_minutes': 0.033,  # Time difference between group mid and transit
    'subband_count': 16,
    'files': [...],  # List of 16 HDF5 file paths
    'days_ago': 9,
    'has_ms': False
}

    Step 3: Find Transit-Centered Window¶

    Location: dsa110_contimg/mosaic/orchestrator.py

    Function: find_transit_centered_window(calibrator_name, timespan_minutes)

    Process:

    1. Get available transits:
    2. Calls calibrator_service.list_available_transits(calibrator_name, max_days_back=60)

    3. Gets list of transits with available data

    4. Select earliest transit:

    5. List is sorted most recent first
    6. Takes last element (earliest transit): earliest_transit = transits[-1]

    7. Extracts transit time: transit_time = Time(transit_iso)

    8. Calculate time window:

    9. Window centered on transit time
    10. Half-span: half_span = TimeDelta(timespan_minutes / 2.0 * 60, format="sec")
    11. Start: start_time = transit_time - half_span

    12. End: end_time = transit_time + half_span

    13. Get calibrator declination:

    14. Loads Dec from catalog: _, dec_deg = calibrator_service._load_radec(calibrator_name)

    15. Count available MS files:

    16. Queries ms_index table for MS files in window: 
      SELECT COUNT(*) FROM ms_index
WHERE mid_mjd >= ? AND mid_mjd <= ?
AND status IN ('converted', 'calibrated', 'imaged', 'done')

    Output: Window info dict:

    {
    'transit_time': Time('2025-11-07T13:19:23'),
    'start_time': Time('2025-11-07T13:13:23'),  # 6 min before transit
    'end_time': Time('2025-11-07T13:25:23'),     # 6 min after transit
    'dec_deg': 55.57,
    'bp_calibrator': '0834+555',
    'ms_count': 3  # Number of MS files in window
}

    Step 4: Process Single Transit (Validation)¶

    Location: dsa110_contimg/conversion/calibrator_ms_service.py

    Function: _process_single_transit(t, calibrator_name, ra_deg, dec_deg, window_minutes, ...)

    Process:

    1. Calculate search window:
    2. t0, t1 = _calculate_transit_window(t, window_minutes)

    3. Window: transit ± (window_minutes / 2)

    4. Query for groups in window:

    5. groups = query_subband_groups(products_db, t0, t1, tolerance_s=1.0)

    6. Finds all 16-subband groups whose observation time overlaps the window

    7. Find best candidate group:

    8. Calls _find_best_candidate_group(groups, t)
    9. Selects group whose mid-time is closest to transit time

    10. Returns: (delta_minutes, best_group, group_mid_time)

    11. Validate complete subband group:

    12. Checks if group has all 16 subbands: _is_complete_subband_group(best_group)

    13. If incomplete, rejects transit

    14. Validate primary beam response:

    15. Calls _validate_primary_beam(group_file, ra_deg, dec_deg, min_pb_response, freq_ghz)
    16. Calculates primary beam response at calibrator position
    17. Requires pb_response >= min_pb_response (default: 0.3 = 30%)

    18. If below threshold, rejects transit

    19. Build result dict:

    20. If all validations pass, returns transit info dict

    Validation Criteria:

    • ✅ Group exists in time window
    • ✅ Group has all 16 subbands
    • ✅ Primary beam response ≥ 30% at calibrator position
    • ✅ Group mid-time closest to transit

    Step 5: Match Transit to Group (Detailed)¶

    Location: dsa110_contimg/conversion/calibrator_ms_service.py

    Function: _match_transit_to_group(transit, group_files, group_id, dec_deg, ...)

    Process:

    1. Parse group start time:
    2. Group ID is timestamp: group_start = Time(group_id)

    3. Example: group_id = "2025-11-07T13:19:23"

    4. Extract group metadata:

    5. Reads HDF5 file to get actual mid-time and declination
    6. group_mid, pt_dec_deg = _extract_group_time_and_dec(group_files, group_id)

    7. Opens first HDF5 file and reads metadata

    8. Validate transit in window:

    9. Checks if transit falls within group's observation window
    10. Window: [group_start, group_start + filelength] (typically 5 minutes)

    11. Transit must be within this window

    12. Check declination match:

    13. Compares group declination to expected calibrator declination
    14. Warning only (doesn't filter): Logs if difference > tolerance

    15. This is why you see warnings like: 

      WARNING - Group 2025-11-07T13:19:23 transit time matches but declination mismatch:
file dec=50.18°, expected 55.57° (diff=5.39°). Trusting transit time match.

    16. Validate complete subband group:

    17. Ensures all 16 subbands are present

    18. Files must be sorted by subband number

    19. Calculate time difference:

    20. dt_min = abs((group_mid - transit).to(u.min).value)
    21. Time difference between group mid-time and transit time

    Output: Transit match dict or None:

    {
    'group_id': '2025-11-07T13:19:23',
    'group_mid': Time('2025-11-07T13:19:25'),
    'group_files_sorted': [...],  # 16 files sorted by subband
    'delta_minutes': 0.033  # How close group mid is to transit
}

    Step 6: Validate Transit in Window¶

    Location: dsa110_contimg/conversion/calibrator_ms_service.py

    Function: _validate_transit_in_window(group_id, transit, filelength)

    Process:

    1. Parse group start time: group_start = Time(group_id)
    2. Calculate group end time: group_end = group_start + filelength
    3. Check overlap:
    4. Transit must fall within [group_start, group_end]
    5. Uses overlap logic: (group_start <= transit) and (transit <= group_end)

    Purpose: Ensures the transit actually occurred during the observation window.

    Step 7: Primary Beam Validation¶

    Location: dsa110_contimg/conversion/calibrator_ms_service.py

    Function: _validate_primary_beam(group_file, ra_deg, dec_deg, min_pb_response, freq_ghz)

    Process:

    1. Read pointing from HDF5 file:
    2. Extracts telescope pointing (RA, Dec) from file metadata

    3. Gets frequency information

    4. Calculate angular separation:

    5. Distance between calibrator position and pointing center

    6. separation = angular_distance(calibrator_ra, calibrator_dec, pointing_ra, pointing_dec)

    7. Calculate primary beam response:

    8. Uses primary beam model (typically Gaussian or Airy pattern)
    9. Response depends on:
      • Angular separation from pointing center
      • Frequency (affects beam size)

    10. Formula: pb_response = f(separation, freq_ghz)

    11. Check threshold:

    12. Requires pb_response >= min_pb_response (default: 0.3 = 30%)
    13. If below threshold, calibrator is too far from pointing center

    Purpose: Ensures calibrator is actually in the primary beam (not just near transit time).

    Step 8: Find Best Candidate Group¶

    Location: dsa110_contimg/conversion/calibrator_ms_service.py

    Function: _find_best_candidate_group(groups, transit)

    Process:

    1. For each group:
    2. Extract group mid-time

    3. Calculate time difference: dt_min = abs((group_mid - transit).to(u.min).value)

    4. Select best:

    5. Group with smallest dt_min (closest to transit)
    6. Returns: (dt_min, best_group, group_mid)

    Purpose: If multiple groups exist in the window, pick the one whose observation time is closest to the transit time.

    Complete Flow Example¶

    Input: create_mosaic_centered_on_calibrator("0834+555", timespan_minutes=12)

    1. Orchestrator calls find_transit_centered_window("0834+555", 12)

    2. Service calls list_available_transits("0834+555", max_days_back=60)

    3. Calculate transits:

    4. RA = 8.58 hours

    5. Previous transits: [2025-11-07T13:19:23, 2025-11-06T13:22:18, ...]

    6. For each transit, match to groups:

    7. Transit: 2025-11-07T13:19:23
    8. Query groups in window: ±30 minutes
    9. Find group: 2025-11-07T13:19:23
    10. Validate: ✅ Complete (16 subbands), ✅ PB response > 30%

    11. Match: delta_minutes = 0.033

    12. Select earliest transit: 2025-11-07T13:19:23

    13. Calculate window:

    14. Transit: 2025-11-07T13:19:23

    15. Window: 2025-11-07T13:13:23 to 2025-11-07T13:25:23 (12 minutes)

    16. Find MS files in window:

    17. Query ms_index for MS files with mid_mjd in window

    18. Count: 3 MS files

    19. Return window info:

    {
    'transit_time': Time('2025-11-07T13:19:23'),
    'start_time': Time('2025-11-07T13:13:23'),
    'end_time': Time('2025-11-07T13:25:23'),
    'ms_count': 3
}
    1. Create mosaic using MS files in this window

    Key Points¶

    1. Transit time is theoretical: Calculated from RA and LST, not from actual observations
    2. Time matching is primary: Transit time must fall within group's observation window
    3. Declination mismatch is warning only: System trusts transit time match even if declination differs
    4. Primary beam validation is critical: Ensures calibrator is actually observable
    5. Complete groups required: All 16 subbands must be present
    6. Earliest transit selected: For mosaic creation, uses the oldest available transit

    Common Warnings Explained¶

    Warning: "Group 2025-11-07T13:19:23 transit time matches but declination mismatch: file dec=50.18°, expected 55.57°"

    Meaning:

    • Transit time matches (transit occurred during observation)
    • But pointing declination (50.18°) differs from calibrator declination (55.57°)
    • System trusts the transit time match and proceeds
    • This can happen if:
    • Telescope was pointing at a different declination
    • Calibrator is still in primary beam (5.39° separation is acceptable)
    • Observation was for a different source but transit time matches

    Action: System continues with the match, trusting transit time over declination.
