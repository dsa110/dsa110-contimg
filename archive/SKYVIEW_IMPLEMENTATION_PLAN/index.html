    SkyView Implementation Plan¶

    Overview¶

    The SkyView page will provide an interactive astronomical image viewer for DSA-110 continuum imaging pipeline products. This will enable users to visualize radio images, overlay catalogs, navigate coordinates, and perform basic measurements.

    User Requirements Analysis¶

    Primary Use Cases¶

    1. Quick Image Inspection
    2. View recently created images
    3. Check image quality (noise, beam, coverage)

    4. Verify calibration and imaging results

    5. Source Verification

    6. Overlay known catalogs (NVSS, VLASS) to verify detections
    7. Compare pipeline images with reference catalogs

    8. Identify calibrator sources

    9. Coordinate Navigation

    10. Navigate to specific RA/Dec coordinates
    11. Center on known sources

    12. Explore different regions of the sky

    13. Image Comparison

    14. Compare images from different time periods
    15. Compare different imaging parameters

    16. View before/after calibration

    17. Quality Assessment

    18. Visual inspection of image artifacts
    19. Check primary beam correction
    20. Verify mosaic coverage

    Key Features Needed¶

    1. Image Display
    2. FITS image rendering (CASA images converted to FITS)
    3. Pan and zoom controls
    4. Colormap selection (grayscale, viridis, plasma, etc.)
    5. Stretch/scale controls (linear, log, sqrt, asinh)
    6. Coordinate grid overlay

    7. WCS coordinate display (RA/Dec)

    8. Image Selection

    9. Browse available images from products database
    10. Filter by date, MS path, image type
    11. Recent images list

    12. Search by path or MS name

    13. Catalog Overlays

    14. NVSS catalog overlay (radio sources)
    15. VLASS catalog overlay (if available)
    16. Custom source lists
    17. Toggle visibility

    18. Source labels with flux information

    19. Measurement Tools

    20. Cursor position display (RA/Dec, pixel coordinates)
    21. Flux measurement at cursor
    22. Distance/angle measurement between points

    23. Region selection (circles, rectangles)

    24. Metadata Display

    25. Image properties panel:

      • Beam size (major, minor, PA)
      • RMS noise
      • Peak flux
      • Image center coordinates
      • Image size (degrees, pixels)
      • Primary beam correction status
      • Creation timestamp
      • Associated MS path

    26. Export/Sharing

    27. Export current view as PNG
    28. Download FITS file
    29. Share image link (with coordinates/zoom state)

    Technical Implementation Plan¶

    Phase 1: Foundation (Core Display)¶

    1.1 Choose Image Viewer Library¶

    Option A: JS9 (JavaScript FITS Viewer) - Pros: - Mature, well-maintained - Excellent FITS support - Good performance - Extensive features (regions, analysis) - Cons: - Requires server-side FITS file serving - Less astronomy-specific UI

    Option B: Aladin Lite - Pros: - Astronomy-focused (CDS) - Built-in catalog support - WCS handling - Good for overlays - Cons: - Heavier dependency - More complex integration

    Recommendation: JS9 - Better for radio astronomy workflows - More flexible for custom features - Easier to integrate with React - Good documentation

    1.2 Backend API Endpoints¶

    New Endpoints Needed:

    1. GET /api/images - List available images
    2. Query params: limit, offset, ms_path, type, pbcor, date_from, date_to

    3. Returns: ImageList with metadata

    4. GET /api/images/{image_id}/metadata - Get detailed image metadata

    5. Returns: Full image properties (beam, noise, WCS, etc.)

    6. GET /api/images/{image_id}/fits - Serve FITS file

    7. Converts CASA image to FITS if needed (on-demand)
    8. Caches converted FITS files

    9. Returns FITS file stream

    10. GET /api/images/{image_id}/thumbnail - Get thumbnail

    11. Returns PNG thumbnail for quick preview

    12. POST /api/images/export-fits - Convert CASA image to FITS

    13. Request: { "image_path": "..." }

    14. Returns: FITS file path or error

    15. GET /api/catalogs/nvss - Query NVSS catalog

    16. Query params: ra, dec, radius_deg
    17. Returns: Catalog sources in region

    Phase 2: Core Features¶

    2.1 Image Loading and Display¶

    Implementation Steps:

    1. Install JS9: 

      npm install js9
# Or use CDN in index.html

    2. Create JS9 wrapper component: 

      // components/SkyViewer.tsx
import { useEffect, useRef } from 'react';
import JS9 from 'js9';

export function SkyViewer({ imagePath }: { imagePath: string }) {
  const containerRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (!containerRef.current || !imagePath) return;

    // Initialize JS9
    JS9.Init(containerRef.current);

    // Load FITS file
    JS9.Load(imagePath);

    return () => {
      JS9.Close();
    };
  }, [imagePath]);

  return <div ref={containerRef} style={{ width: '100%', height: '600px' }} />;
}

    3. Add FITS file serving endpoint:

    4. Check if FITS exists
    5. If not, convert CASA image to FITS (async)
    6. Return FITS file path
    7. Cache converted files

    2.2 Image Browser¶

    Implementation Steps:

    1. Create API hook: 

      // api/queries.ts
export function useImages(filters?: ImageFilters) {
  return useQuery({
    queryKey: ['images', filters],
    queryFn: async () => {
      const response = await apiClient.get<ImageList>('/api/images', { params: filters });
      return response.data;
    },
  });
}

    API Models¶

    # api/models.py
class ImageInfo(BaseModel):
    id: int
    path: str
    ms_path: str
    created_at: datetime
    type: str  # "image", "pbcor", "residual", "psf", "pb"
    beam_major_arcsec: Optional[float]
    beam_minor_arcsec: Optional[float]
    beam_pa_deg: Optional[float]
    noise_jy: Optional[float]
    peak_flux_jy: Optional[float]
    pbcor: bool
    center_ra_deg: Optional[float]
    center_dec_deg: Optional[float]
    image_size_deg: Optional[float]
    pixel_size_arcsec: Optional[float]

class ImageList(BaseModel):
    items: List[ImageInfo]
    total: int

class ImageMetadata(BaseModel):
    image: ImageInfo
    wcs: Optional[Dict]  # WCS header information
    stats: Optional[Dict]  # Image statistics
