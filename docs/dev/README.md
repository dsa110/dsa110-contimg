# Development Documentation

**Purpose:** Development notes, status reports, analysis, and history  
**Audience:** Developers, AI agents, maintainers

---

## Structure

This directory contains development-related documentation organized by type:

### [`analysis/`](analysis/)
Investigation reports, bug analysis, technical deep-dives, and root cause analyses.

**Examples:**
- Time handling issues analysis
- RA calculation bug investigation
- Performance analysis
- Unanticipated issues discovered during development

### [`status/`](status/)
Temporal status reports organized by month (YYYY-MM/). These documents track progress, blockers, and current state of features or components.

**Examples:**
- Dashboard implementation status
- Testing summaries
- Endpoint verification results
- Sprint completion reports

**Note:** Status reports older than 1 year should be archived to `docs/archive/status_reports/`.

### [`notes/`](notes/)
Agent notes, brainstorming documents, design considerations, and implementation notes.

**Examples:**
- Architectural brainstorming
- Design considerations
- Implementation notes
- Optimization ideas

**Note:** Periodically review notes and extract valuable insights to permanent documentation.

### [`history/`](history/)
Completed investigations, development history, and changelog entries.

**Examples:**
- Completed analysis reports (moved from `analysis/` when done)
- Development history summaries
- Changelog entries
